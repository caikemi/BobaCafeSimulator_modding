# UE5.6 Model-Asset Mod PAK Packaging Guide

[中文](Model_PAK_Packaging_ZH.md) | [English](Model_PAK_Packaging_EN.md)

> [!IMPORTANT]
> This guide was translated with AI and may contain inaccurate wording. If anything is unclear or conflicts with actual Unreal Engine or game behavior, refer to the [original Chinese guide](Model_PAK_Packaging_ZH.md).

This guide applies to Windows model-asset Mods for "Boba Cafe Simulator - Reborn as the Manager of Ice Castle Sweet City."

When finished, a model-asset Mod will normally contain:

```text
MyDecorationMod/
├── main.lua
├── MyDecorationMod.pak
├── ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM5-PCD3D_SM5.ushaderbytecode
├── ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM6-PCD3D_SM6.ushaderbytecode
├── preview.png
└── SM_MyDecoration.png
```

> `main.lua` and preview PNG files are ordinary loose files and do not belong in the PAK. Models, materials, material instances, and textures must be cooked by UE and written into the PAK.

## 1. Create a blank UE5.6 project

Use the same engine version as the game: **Unreal Engine 5.6**.

1. Open UE5.6.
2. Select **Games**.
3. Select the **Blank** template.
4. Choose a **Blueprint** project.
5. Choose **Desktop** as the target platform.
6. Use English for both the project name and project path, for example:

```text
MyDecorationProject
```

Use a reasonably unique project name. Both the project name and Chunk ID appear in the ShaderArchive filename, so an independent name reduces collisions between Mods.

Cooked content depends on both the engine version and target platform. Assets packaged with another UE version are not guaranteed to load in the current game.

## 2. Understand the project directories

A newly created blank project has approximately this structure:

```text
MyDecorationProject/
├── MyDecorationProject.uproject
├── Config/
│   ├── DefaultEngine.ini
│   └── DefaultGame.ini
├── Content/
│   └── MyDecorationPack/
│       ├── PAL_MyDecorationPack.uasset
│       ├── SM_MyDecoration.uasset
│       ├── M_MyDecoration.uasset
│       ├── MI_MyDecoration.uasset
│       └── T_MyDecoration_Color.uasset
└── Saved/
    └── StagedBuilds/
```

What each location contains:

| Location | Contents |
|---|---|
| Project root | The `.uproject` file |
| `Config/` | Project packaging, rendering-platform, and Asset Manager settings |
| `Content/` | Every asset visible in the UE Content Browser |
| `Content/MyDecorationPack/` | This Mod's models, materials, textures, and PrimaryAssetLabel |
| `Saved/StagedBuilds/` | PAK files generated by Cook and Package |
| The game's `BobaCafeSimulator/Mods/` | Final installation and runtime location for a Mod; this is not a UE project asset directory |

Create a dedicated English-named directory under `Content` for every Mod:

```text
Content/MyDecorationPack/
```

Its corresponding runtime UE path is:

```text
/Game/MyDecorationPack/
```

The directory name is embedded in cooked assets and in the PAK. Renaming the PAK after packaging does not change its internal asset paths.

## 3. Organize models, materials, and textures

Put every asset used by the Mod under its own asset directory:

```text
Content/MyDecorationPack/
├── SM_MyDecoration
├── M_MyDecoration
├── MI_MyDecoration
├── T_MyDecoration_Color
├── T_MyDecoration_Normal
├── T_MyDecoration_Roughness
└── PAL_MyDecorationPack
```

Before packaging, verify that:

- Every Static Mesh material slot references the correct material or material instance.
- Each material instance references the correct parent material.
- Every color, normal, roughness, and metallic texture referenced by a material exists.
- All models, materials, and textures have been saved.
- There are no unresolved redirectors in the Content Browser.

If the PAK contains only `SM_*.uasset` and no materials or textures, the model may appear black, use the default material, or fail to load.

## 4. Create and place a PrimaryAssetLabel

A `PrimaryAssetLabel` is a **UE data asset** that tells the Asset Manager:

- Which assets must be cooked.
- Which Chunk owns those assets.
- Which dependencies must follow them into the corresponding PAK.

It is not a configuration file or a folder. It should appear in the UE Content Browser.

Open this folder in the Content Browser:

```text
Content/MyDecorationPack/
```

Then:

1. Right-click an empty area in that folder.
2. Select **Data Asset** under **Miscellaneous** or **Advanced Asset**.
3. Select `PrimaryAssetLabel` as the data-asset class.
4. Name it:

```text
PAL_MyDecorationPack
```

Its location on disk is:

```text
ProjectRoot/Content/MyDecorationPack/PAL_MyDecorationPack.uasset
```

Its UE asset path is:

```text
/Game/MyDecorationPack/PAL_MyDecorationPack
```

Keep `PAL_MyDecorationPack` in the same folder as the models, materials, and textures it manages. If the model uses subfolders, `Label Assets in My Directory` also manages them recursively.

Open `PAL_MyDecorationPack` and configure:

| Option | Recommended value | Purpose |
|---|---:|---|
| `Priority` | `1` | Priority of the Chunk rules |
| `Chunk ID` | `1001` | Generates `pakchunk1001-Windows.pak` |
| `Cook Rule` | `Always Cook` | Forces managed assets to be cooked |
| `Label Assets in My Directory` | Enabled | Manages the current directory and all subdirectories |
| `Is Runtime Label` | Enabled | Makes the label a runtime Primary Asset |

Do not create a Blueprint subclass of `PrimaryAssetLabel`. Create a `PrimaryAssetLabel` data asset directly.

Different asset packs may use different Chunk IDs, such as `1001`, `1002`, and `1101`. Do not use Chunk `0`, because project content that is not assigned to another Chunk enters `pakchunk0` by default.

Epic documentation about PrimaryAssetLabel and Chunks:

- [Cooking Content and Creating Chunks](https://dev.epicgames.com/documentation/en-us/unreal-engine/cooking-content-and-creating-chunks-in-unreal-engine)
- [Preparing Assets for Chunking](https://dev.epicgames.com/documentation/en-us/unreal-engine/preparing-assets-for-chunking-in-unreal-engine)

## 5. Verify the Asset Manager

Open:

```text
Edit → Project Settings → Game → Asset Manager
```

Confirm that `Primary Asset Types to Scan` contains `PrimaryAssetLabel` and that its scan directories include:

```text
/Game
```

A blank project can normally discover a `PrimaryAssetLabel` without extra configuration. If the newly created Chunk does not appear under **Tools → Audit → Asset Audit**, check this setting first.

Open:

```text
Tools → Audit → Asset Audit
```

Click `Add Chunks` and confirm that Chunk `1001` contains:

- `PAL_MyDecorationPack`
- The Static Mesh
- Materials and material instances
- Textures

## 6. Configure Windows shader formats

Open:

```text
Edit → Project Settings → Platforms → Windows
```

Use settings that match the game:

```text
Default RHI                    = DirectX 12
D3D12 Targeted Shader Formats = SM5, SM6
D3D11 Targeted Shader Formats = SM5
```

Packaging will then normally generate both SM5 and SM6 ShaderArchives. The game opens the file corresponding to the player's current RHI.

## 7. Configure Packaging

Open:

```text
Edit → Project Settings → Project → Packaging
```

Configure:

| Option | Value |
|---|---:|
| `Use Pak File` | Enabled |
| `Use Io Store` | **Disabled** |
| `Use Zen Store` | Disabled |
| `Generate Chunks` | Enabled |
| `Create compressed cooked packages` | Enabled |
| `Share Material Shader Code` | **Enabled** |
| `Shared Material Native Libraries` | **Disabled** |
| `Cook everything in the project content directory` | Disabled |

The current Mod loader mounts `.pak` files, not `.utoc/.ucas`, so the Mod authoring project must disable IoStore.

This compatibility requirement also applies to the base game:

- Mod authors must disable IoStore in the UE project used to cook assets, producing a traditional `.pak` and shared ShaderArchive.
- The game developer must also disable IoStore when releasing `BobaCafeSimulator`. If the base game still uses IoStore at runtime, models may load but a pure-PAK Mod's `.ushaderbytecode` cannot be opened as a traditional shared shader library. The material will then appear as the default gray or black.

Regular Mod authors do not need to change the installed game's configuration. Use an official game build that supports traditional PAK Mods and make sure IoStore is disabled in your own authoring project.

`Share Material Shader Code` must be enabled. This writes material shaders to separately loadable ShaderArchives and avoids compatibility problems caused by loading inline cooked shaders directly in Editor/PIE.

Add the following to **Additional Asset Directories to Cook**:

```text
/Game/MyDecorationPack
```

The relevant part of `Config/DefaultGame.ini` will look like:

```ini
[/Script/UnrealEd.ProjectPackagingSettings]
UsePakFile=True
bUseIoStore=False
bUseZenStore=False
bGenerateChunks=True
bCompressed=True
bShareMaterialShaderCode=True
bSharedMaterialNativeLibraries=False
bCookAll=False
+DirectoriesToAlwaysCook=(Path="/Game/MyDecorationPack")
```

If this section already exists, edit its existing fields. Do not create multiple configuration sections with the same name.

Epic documentation about Pak, IoStore, and Generate Chunks:

- [Packaging Your Project](https://dev.epicgames.com/documentation/en-us/unreal-engine/packaging-your-project)
- [Project Packaging Settings](https://dev.epicgames.com/documentation/en-us/unreal-engine/project-section-of-the-unreal-engine-project-settings)

## 8. Cook and package a Windows PAK

### Method 1: package from the Editor

From the UE5.6 top menu, select:

```text
Platforms → Windows → Package Project
```

Clicking **Cook Content** alone does not complete PAK packaging. Run Package to output a separate Chunk PAK.

### Method 2: use PowerShell

Replace these two paths with the locations of your own UE5.6 installation and blank project:

```powershell
$UE_ROOT = "C:\Path\To\UE_5.6"
$PROJECT = "C:\Path\To\MyDecorationProject\MyDecorationProject.uproject"

& "$UE_ROOT\Engine\Build\BatchFiles\RunUAT.bat" `
  "-ScriptsForProject=$PROJECT" `
  BuildCookRun `
  -nop4 `
  -utf8output `
  -nocompileeditor `
  -skipbuildeditor `
  -cook `
  "-project=$PROJECT" `
  "-unrealexe=$UE_ROOT\Engine\Binaries\Win64\UnrealEditor-Cmd.exe" `
  -platform=Win64 `
  -installed `
  -stage `
  -package `
  -clean `
  -pak `
  -compressed `
  -manifests `
  -nocompile `
  -nocompileuat
```

Use a new empty output directory for every rebuild, or clean old Stage/Package output first. Do not simply overwrite the new `.pak` files from an IoStore-disabled build into an older release directory that was built with IoStore, because packaging does not necessarily delete old files.

In particular, when the game developer releases the base game, confirm that the final:

```text
BobaCafeSimulator/Content/Paks/
```

contains none of the following leftovers:

```text
*.utoc
*.ucas
global.utoc
global.ucas
```

Updating only `.pak` while leaving those old files causes runtime to remain in IoStore mode. Mod models may load while their material shader libraries cannot open. Always package to a new empty directory before submitting the Steam Depot.

After a successful package, the PAK is normally located at:

```text
ProjectRoot/
└── Saved/
    └── StagedBuilds/
        └── Windows/
            └── MyDecorationProject/
                └── Content/
                    └── Paks/
                        ├── pakchunk0-Windows.pak
                        └── pakchunk1001-Windows.pak
```

Copy the Chunk specified in the PrimaryAssetLabel:

```text
pakchunk1001-Windows.pak
```

Do not use `pakchunk0-Windows.pak` as the model Mod.

You may rename the Chunk PAK to something easier to recognize:

```text
MyDecorationMod.pak
```

Only rename the PAK file itself. Do not change its internal directory structure.

## 9. Extract ShaderArchives from the PAK

The shared shader files are written into the PAK, but after UE5.6 Editor dynamically mounts a new PAK, it cannot reliably open a newly introduced ShaderArchive directly from that PAK.

Extract the corresponding `.ushaderbytecode` files as loose files and place them in the final Mod directory as well.

PowerShell example:

```powershell
$UE_ROOT = "C:\Path\To\UE_5.6"
$PAK = "C:\Path\To\pakchunk1001-Windows.pak"
$OUTPUT = "C:\Path\To\ExtractedShaders"

New-Item -ItemType Directory -Force -Path $OUTPUT

& "$UE_ROOT\Engine\Binaries\Win64\UnrealPak.exe" `
  $PAK `
  -Extract $OUTPUT `
  '-Filter=ShaderArchive-*.ushaderbytecode'
```

Normally this produces:

```text
ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM5-PCD3D_SM5.ushaderbytecode
ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM6-PCD3D_SM6.ushaderbytecode
```

Rules:

- You may rename the PAK.
- Do not rename a ShaderArchive.
- Distribute both the SM5 and SM6 files with the Mod.
- Do not copy only the PAK and omit the ShaderArchives.

## 10. Write the `main.lua` asset path

If the mesh is located at:

```text
Content/MyDecorationPack/SM_MyDecoration.uasset
```

Its runtime object path is:

```text
/Game/MyDecorationPack/SM_MyDecoration.SM_MyDecoration
```

Lua example:

```lua
local ASSET_ROOT = "/Game/MyDecorationPack"
local MeshName = "SM_MyDecoration"
local MeshPath = ASSET_ROOT .. "/" .. MeshName .. "." .. MeshName

local Mesh = MOD.GAA.LoadObject("StaticMesh'" .. MeshPath .. "'")
if not Mesh then
    error("Mesh failed to load: " .. MeshPath)
end
```

The path comes from the directory under the UE project's `Content`, not from the PAK filename or final Mod folder name.

The complete example includes PAK mounting, registration of four furniture items, preview images, and English/Chinese text:

- [Complete AnimalDecorationAssetPack example](Example_EN/AnimalDecorationAssetPack/)
- [View `main.lua`](Example_EN/AnimalDecorationAssetPack/main.lua)

## 11. Assemble the final Mod folder

Recommended structure:

```text
MyDecorationMod/
├── main.lua
├── MyDecorationMod.pak
├── ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM5-PCD3D_SM5.ushaderbytecode
├── ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM6-PCD3D_SM6.ushaderbytecode
├── preview.png
├── SM_MyDecoration.png
└── SM_AnotherDecoration.png
```

File purposes:

| File | Purpose |
|---|---|
| `main.lua` | Mount the PAK, load meshes, and register furniture data |
| `*.pak` | Cooked models, materials, and textures |
| `*.ushaderbytecode` | Shared material shader libraries |
| `preview.png` | Preview image for the Mods menu or Steam Workshop |
| One PNG per furniture item | Individual preview image shown in the furniture store |

Place the entire folder at:

```text
GameRoot/BobaCafeSimulator/Mods/MyDecorationMod/
```

Then enable it in the game's Mods menu.

## 12. Inspect the PAK contents

Use UnrealPak to list the files:

```powershell
$UE_ROOT = "C:\Path\To\UE_5.6"
$PAK = "C:\Path\To\MyDecorationMod.pak"

& "$UE_ROOT\Engine\Binaries\Win64\UnrealPak.exe" $PAK -List
```

You should see content similar to:

```text
MyDecorationPack/SM_MyDecoration.uasset
MyDecorationPack/SM_MyDecoration.uexp
MyDecorationPack/M_MyDecoration.uasset
MyDecorationPack/MI_MyDecoration.uasset
MyDecorationPack/T_MyDecoration_Color.uasset
ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM5-PCD3D_SM5.ushaderbytecode
ShaderArchive-MyDecorationProject_Chunk1001-PCD3D_SM6-PCD3D_SM6.ushaderbytecode
```

Some assets also include `.ubulk` files. This is normal.

Also verify that the mount point shown near the beginning of `UnrealPak -List` includes:

```text
<ProjectName>/Content/
```

The common Mod loader preserves the relative directory after `Content` and maps it to the game's `/Game/...` path.

## 13. Troubleshooting

### The PAK mounts successfully, but the model does not exist

Check:

- Whether `ASSET_ROOT` matches the asset directory under `Content`.
- Whether `MeshName` exactly matches the UE asset name.
- Whether the `PrimaryAssetLabel` specifies the correct Chunk.
- Whether you accidentally copied `pakchunk0`.

### The model appears without its material

Check:

- The Static Mesh material slots.
- Whether the materials and textures entered the same Chunk PAK.
- Whether `Share Material Shader Code` is enabled.
- Whether both ShaderArchives are in the Mod root.
- Whether IoStore is disabled in the released base game.
- Whether old `.utoc/.ucas` files remain under the base game's `BobaCafeSimulator/Content/Paks/`.

When a shared shader library loads normally, the game log should contain lines similar to:

```text
Using ...ShaderArchive-...ushaderbytecode
Logical shader library '...' has been created
成功打开 Mod Shader 库
```

The last line is the literal Chinese runtime log for "Mod shader library opened successfully."

If you see `打开 Mod Shader 库失败` ("failed to open Mod shader library"), `Missing shader resource`, `Tried to access an uncooked shader map ID`, or the startup log still reads `global.utoc`, first check whether the base game still uses IoStore or mixes output from old and new packages.

### Packaging produces only `.utoc` and `.ucas`

`Use Io Store` is still enabled. The current Mod loader requires a `.pak`.

### RenderCore reports an array out-of-bounds error or `Missing shader resource`

This normally means:

- Inline cooked shaders were used.
- Shared Shader Code was not enabled.
- The ShaderArchives were not extracted to the Mod directory.
- A ShaderArchive was renamed.

### Editor/PIE reports Unversioned Content

During PIE initialization, the complete example executes:

```text
s.AllowUnversionedContentInEditor 1
```

This setting is only for Editor/PIE testing. You do not need to enter it manually when a packaged game loads a cooked Mod.
